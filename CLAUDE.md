## 앱 철학

광맥은 주식을 도박·투기가 아닌 **'미래를 위한 건강한 투자'로 체득**하게 하는 앱이다.
사용자가 포트폴리오를 차근차근 가꾸고, 점수를 높이며, 누적금액이 늘어나는
**'내 돈이 돈을 버는 구조'를 시각적으로 보고 느끼고, 실제 기록으로 스스로 알아가게** 한다.

종목 추천 앱이 **절대 아니다**.
모든 평가·점수는 사용자가 **자기 렌즈로 자기 포트를 비추는 것**이지,
무엇을 사라는 게 아니다.

사용자가 종목을 **공부하고(재무·비전 이해) 자기 미래관과 같은 방향이라 판단해 매수해가도록** 돕는다.
충동·추천이 아닌 **이해 기반 투자**.

## 0. 한 줄 정의
키움 연동 기반, 한국 장기·배당 투자자를 위한 **게임형 포트폴리오 트래커**.
컨셉: *"잠자는 동안 캐는 내 포트폴리오 금광."*

## 1. 핵심 가치 (왜 쓰는가 — 아직 가설. FM코리아 검증 필요)
1. **기록 노가다 제거** — 매매·배당을 자동 기록(엑셀 손기록 대체). ← 진짜 itch
2. **측량소 점수(2모드)** — 기본: 내 포트의 객관 펀더멘털 스냅샷.
   테마 설정 시: 미래정합성(내가 고른 방향과 정렬도). ← 차별점·자랑거리
3. **게임화** — 금광 메타포로 "열고 싶은 앱". ← 리텐션
4. **공유** — FM코리아 수익인증 문화에 맞춘 한 장 카드. ← 분배

## 3. 기술 스택
- **프론트**: React + Vite
- **백엔드**: FastAPI (키움 앱키·시크릿 서버 보관)
- **DB**: SQLite (v1) → Postgres(추후)
- **배포**: Railway
- **런타임**: **Python 3.12 고정** (3.14 wheel 이슈 회피)
- **데이터**: 키움 REST(국내), DART(재무). 해외주식은 v2.

## 4. 하드 룰 (절대 — 위반 금지)
- 코드 작성 전 **아키텍처 먼저 제안·승인**.
- **매수/매도 조언 생성 금지**(자본시장법). 모든 분석에 "투자권유 아님" 명시.
- 점수·티어는 **스타일 중립**. 특정 투자철학(분산·배당 등) 우대 금지.
- **EXP는 원(₩) 액수에 비례 금지.** 수익률(%)·이벤트 고정값만 허용. **손실로 레벨 강등 금지** (EXP 단조 증가).
- **레벨/EXP는 시장 통합(KR·US 합산) — KR/US 분리 원칙의 유일한 예외.** 포트 데이터(보유·거래·배당)는 여전히 KR/US 분리.
- **스타일 중립**: 3개 EXP 소스(①배당 ②익절 ③행동·v2)가 각 성향(배당형·성장형·장기형)의 레벨업 경로를 동등하게 연다. 특정 투자철학 우대 금지.
- 측량소는 **2모드**: ① 기본 모드(펀더멘털 스냅샷, 디폴트) ② 테마 모드(미래정합성, 사용자 설정 시).
  "미래정합성" 용어는 테마 모드 전용. 두 모드 다 "투자권유 아님 / 점수 낮음 ≠ 결함" 고정.
- **완벽 중립 점수는 없다.** 어떤 지표도 관점을 내포한다.
  해결책은 점수를 없애는 게 아니라 **무슨 기준으로 무엇을 보는지 투명하게 공개**하는 것이다.
- 키·시크릿은 `.env`. git·프론트 노출 금지.
- 데이터 접근은 **함수로 추상화**(mock → 키움 무손실 교체).
- Python 3.12 고정.

## 5. 비주얼 / 광산 씬 아키텍처
**영상(mp4) 아님. 실시간 레이어 합성. 배경은 고정.**

레이어 (뒤→앞):
- **L0 배경** — 고정 페인티드 이미지. 성향·상태 무관 **불변**. `assets/scene/bg_mine.png`
- **L1 광맥 + 금더미** — 모두 **금색**.
  - 광맥 = 금색 수정(고정). `assets/scene/vein_gold.png`
  - 금더미 = 금괴, **크기 3단계(소/중/대)**, 수익에 따라 교체. `assets/scene/gold_s/m/l.png`
- **L2 오로라/글로우** — **금색 고정**(따뜻한 빛). 코드 글로우(셰이더/캔버스).
- **L3 캐릭터·수레·파티클** — 일꾼(채굴 루프), 수레(레일 위, 수확 트윈), 반짝임.
  `assets/scene/miner.png`, `cart_empty.png`, `cart_full.png`
- **L4 UI 오버레이** — HUD·수익률·수확버튼·독.

규칙:
- **성향은 색이 아니라 글씨**로 표시(v1). 광산은 금색 한 톤 통일.
  - (v2 옵션: 성향별 색 스킨 — 흰 수정 1세트를 코드 틴트해 N색. 지금은 보류.)
- **수확 인터랙션**: 탭 → 수레가 레일 따라 금더미로 트윈 이동 → 합체 → 금더미 크기↑ + 숫자 카운트업. **실제 배당 데이터 기반.**
- 에셋은 **한 장에 굽지 말 것** — 배경/광맥/금더미/수레/일꾼 분리 유지(애니·교체용).

### HUD 구성 (확정 — 2026-06 레벨/EXP로 갱신)
- **상단 좌**: 결정(crystal) 모양 **Lv 배지**(`Lv N`, 레벨↑일수록 글로우↑) + "광부 레벨" 라벨.
- **그 아래**: **EXP 채굴 게이지**(현재/필요 EXP, 다음 레벨까지). 시장 통합(KR+US 합산).
- **2단**: 광맥 등급 칩 `광맥 A · 88` (US면 `US 광맥 A · 91`) · 시장 토글 · 성향 칩 `성향 · 공격형`.
- **우상단**: 알림/설정 아이콘.
- **3단**: 총 평가금액 (시장 통화 `₩`/`$`).
- ~~티어 엠블럼·`마스터 4` 디비전~~ → **제거**(레벨/EXP로 대체). 엠블럼 에셋은 보존(v2 활용 여지).
- **수익률 패널**(중앙 상단, 탭 가능): `▲ +18.4% / 평가수익 +₩1.72M`

### 보유 종목 (개별주) — 동적 데이터 리스트
- **종목은 절대 씬 아트에 굽지 않는다.** 보유 배열로 코드가 그리는 칩/리스트. → 매도=칩 빠지고 자동 정렬, 매수=칩 추가. (이게 sell/buy UI 문제의 해법)
- **홈(미리보기)**: 수익률 패널 바로 밑에 **상위 3개만** 칩으로(`삼성전자 +12.4%` 식). 넘으면 숨기고 `›` 표시.
- **펼침(전체)**: 수익률 패널/`›` 탭 → 바텀시트로 **현재 시장의 전 종목** 표시(이름·평가금액·수익률, 스크롤). 씬은 디밍.
- 수익률 색: 이익=초록, 손실=빨강.

### 시장 토글 (KR / US) — v1 포함
- 홈은 **한 번에 한 시장**만 본다(KR 또는 US). 상단 중앙 버튼으로 전환.
- 버튼 라벨: KR 보는 중 → `US 광맥 이동`, 누르면 US로 전환 후 버튼 → `KR 광맥 이동`.
- 전환 시 **씬(광산 그림)은 동일**, 데이터만 교체: 수익률·개별주·금괴 금액·**통화(₩↔$)**·등급 라벨(`광맥 A`↔`US 광맥 A`).
- **KR/US 완전 분리** — 각자의 광맥 등급·금더미·티어. 환율 합산 안 함. (통합 보기는 추후 별도 기능)

## 6. 점수 / 등급 로직 (요약 — 상세 산식은 `docs/scoring_v2_design.md`)

### 공통 원칙
- 측량소 점수는 측량소 전용. 티어/XP와 무관. 매수/매도 조언 금지.
- 완벽 중립 점수는 없다. 무슨 기준으로 보는지 **투명 공개로 중립성을 대체**한다.

### 성향 분류 (변경 없음)
변동성·섹터집중도·성장vs배당 비중·현금비중·종목수 → 4유형.
30일 이동평균 + 히스테리시스(잦은 전환 방지). **규모 무관.**

### 측량소 점수 — 2모드

**① 기본 모드 (디폴트) — "펀더멘털 스냅샷"**
처음 들어온 사용자가 보는 기본 점수. "우열 평가"가 아닌 **포트폴리오 건강검진·사실 스냅샷**.
- 산식: `fundᵢ = 0.5·health_i + 0.3·supply_i + 0.2·trust_i` (N/A 항목 재정규화)
  - `health(재무건전성)` = 수익성·안정성·성장성 3하위축 균형.
    **성장의 실효비중 = 전체의 15% 상한** (단일 철학 지배 방지. 하드룰 §4 충돌 해소).
  - `supply(수급)` = 외인+기관 20일 순매수비율 → tanh 정규화 (ka10059, 확인 완료).
  - `trust(시장신뢰)` = 시총·거래대금 로그 백분위 (ka10001 — **필드 검증이 선결 게이트**.
    미확보 시 이 축 드롭 후 재정규화).
- 총점: `S = Σ wᵢ·fundᵢ`, 기여도 `cᵢ = wᵢ·fundᵢ` (Σcᵢ = S 정확 분해).
- HUD 라벨: `광맥 A · 88`
- UI 문구 고정: "투자권유 아님 / 점수 낮음 ≠ 나쁜 포트폴리오 / 우열이 아닌 사실 스냅샷"

**② 테마 모드 (사용자가 직접 설정 시) — "미래정합성"**
"내가 고른 방향성에 내 포트가 얼마나 정렬됐나." **"미래정합성"은 이 모드 전용 용어.**
- 사용자가 테마를 직접 선택/입력. "내 성장테마 삭제하기" → ① 기본 모드 복귀.
- 산식: `sᵢ = alignᵢ · fundᵢ` (정렬 × 펀더멘털. alignᵢ=1|0, fundᵢ는 기본 모드 재사용)
- 화면: **정렬도(%)** + **정렬 종목 품질** 분리 노출 → "비중 부족인지 종목이 약한지" 구분.
- 테마 매칭 3단 폴백:
  **(A) 큐레이션 바스켓**(JSON 분리, "참고용 큐레이션, 절대 기준 아님" 명시)
  \> (B) DART 업종(KSIC) \> (C) 키워드. 매칭 근거(`matchSource`) 투명 표기.
- HUD 라벨: `AI테마 정렬 A · 88` (설정된 테마명 부기)
- UI 문구 고정: "투자권유 아님 / 사용자가 설정한 렌즈로 자기 포트를 비추는 것 / 종목 추천 아님"

### 티어 / XP (변경 없음)
배치(스타일 중립, 상한 골드) → 등반(행동·비율·연속 XP). **액수 비례 금지, 손실 무강등.** 디비전 IV→I.

### 탐사(what-if) (변경 없음)
종목명 입력 → 가상 비중으로 **활성 모드** 점수 재계산 → 등급 변화 시뮬. **"가정 시뮬·투자권유 아님" 고정.**

## 7. 화면 (5개, 하단 독)
- **광산(홈)**: 금광 씬 + HUD + 수익률 패널 + **개별주 상위 3개 미리보기** + **KR/US 시장 토글**. 감성·글랜스.
- **금고**: 매매·배당 자동 기록(영구 장부), 월/연 조회, 수확, Excel/PDF Export. **현재 시장 전 종목 리스트**(홈 펼침과 연결). ← v1 핵심
- **측량소**: ① 기본 모드(펀더멘털 스냅샷, 디폴트) / ② 테마 모드(미래정합성, 사용자 설정 시). 종목별 기여도·매칭 근거 투명 분해.
- **탐사**: 종목 **공부 도구 + what-if 시뮬**. 종목 검색 → 회사 소개(어떤 회사·뭘 만드는지·어디 영향력, 2줄 이내)·PER·ROE·PBR·외인·기관 순매수 bar 시각화 한 화면. "재무제표 읽는 습관 형성"이 목적. ★ 지표는 사실+업종평균 비교로만 — 판정·권유 금지, 판단은 사용자 몫. **개인 순매수·미래 비전 한줄은 v2에서 추가.**
- **연구소**: 종목 심층 분석(재무·수급·이슈). v2 이후.

> **US 데이터 주의**: 홈의 KR/US 토글은 v1 UI에 포함하되, **미국 주식 시세·거래·배당 데이터 소스 확보 여부를 먼저 확인**할 것. 키움 해외주식 API 또는 대체 소스가 안 되면, v1은 US를 "준비 중(잠금)" 상태로 두고 KR만 실데이터로 가고, US 실연동은 v2로. (UI/토글 구조는 그대로 두고 데이터만 채우는 식.)

## 13. 감정 설계 철학 (v2 방향의 핵심 원칙)
광맥의 진짜 적은 **"아무 감정도 안 드는 투자앱"**이다.
남은 과제는 기능 추가가 아니라 **"자기 투자 인생을 RPG 캐릭터처럼 느끼게 하는 감정 설계"**.

- **업적·서사 > 랭킹**: 남과 비교하는 랭킹은 오래 못 간다. 업적(첫배당, 100일연속)과 타임라인(어제의 나)이 리텐션을 만든다.
- **검증 우선**: 기능을 더 쌓기 전에 광부 명함으로 실제 반응(FM코리아 등)을 먼저 확인. 반응이 있으면 업적 시스템 본격화.
- **감정의 순서**: 금괴가 커지는 걸 본다 → 업적이 터진다 → 타임라인에 기록된다 → 명함으로 자랑한다. 이 순환이 앱을 열게 만든다.
