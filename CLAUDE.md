## 0. 한 줄 정의
키움 연동 기반, 한국 장기·배당 투자자를 위한 **게임형 포트폴리오 트래커**.
컨셉: *"잠자는 동안 캐는 내 포트폴리오 금광."*

## 1. 핵심 가치 (왜 쓰는가 — 아직 가설. FM코리아 검증 필요)
1. **기록 노가다 제거** — 매매·배당을 자동 기록(엑셀 손기록 대체). ← 진짜 itch
2. **미래정합성 점수** — 내 포트가 성장 테마에 얼마나 정렬됐나. ← 차별점·자랑거리
3. **게임화** — 금광 메타포로 "열고 싶은 앱". ← 리텐션
4. **공유** — FM코리아 수익인증 문화에 맞춘 한 장 카드. ← 분배

## 2. 용어집
| 용어 | 의미 |
|---|---|
| 광산 | 홈 화면 = 내 포트폴리오 전체 |
| 금더미(금괴) | 모은 금 = 누적 자산/수익. **항상 금색, 크기만 커짐** |
| 광맥 | 광산 중앙의 빛나는 금색 수정(금광의 심장). **금색 고정** |
| 수레 | 배당. 수확 시 레일 따라 금더미로 굴러가 합쳐짐 |
| 성향 | 회피/안정/공격/하이브리드. 정체성(우열 아님). **v1은 글씨로 표시** |
| 미래정합성 점수 / 광맥 등급 (A·88) | 성장테마 정렬도. "성장 렌즈"라는 의견. 측량소에만 |
| 티어 | 아이언~챌린저, 디비전 IV~I. 행동·규율 XP(스타일 중립) |

## 3. 기술 스택
- **프론트**: React + Vite
- **백엔드**: FastAPI (키움 앱키·시크릿 서버 보관)
- **DB**: SQLite (v1) → Postgres(추후)
- **배포**: Railway
- **런타임**: **Python 3.12 고정** (3.14 wheel 이슈 회피)
- **데이터**: 키움 REST(국내), DART(재무). 해외주식은 v2.

## 4. 하드 룰 (절대 — 위반 금지)
- 코드 작성 전 **아키텍처 먼저 제안·승인**.
- **매수/매도 조언 생성 금지**(자본시장법). 모든 분석에 "투자권유 아님" 명시.
- 점수·티어는 **스타일 중립**. 특정 투자철학(분산·배당 등) 우대 금지.
- **XP는 원(₩) 액수에 비례 금지.** **손실로 티어 강등 금지.**
- 미래정합성은 **"성장테마 렌즈"** 임을 명시(보편 점수 아님). 배당/가치 포트 낮음 = 결함 아님.
- 키·시크릿은 `.env`. git·프론트 노출 금지.
- 데이터 접근은 **함수로 추상화**(mock → 키움 무손실 교체).
- Python 3.12 고정.

## 5. 비주얼 / 광산 씬 아키텍처
**영상(mp4) 아님. 실시간 레이어 합성. 배경은 고정.**

레이어 (뒤→앞):
- **L0 배경** — 고정 페인티드 이미지. 성향·상태 무관 **불변**. `assets/scene/bg_mine.png`
- **L1 광맥 + 금더미** — 모두 **금색**.
  - 광맥 = 금색 수정(고정). `assets/scene/vein_gold.png`
  - 금더미 = 금괴, **크기 3단계(소/중/대)**, 수익에 따라 교체. `assets/scene/gold_s/m/l.png`
- **L2 오로라/글로우** — **금색 고정**(따뜻한 빛). 코드 글로우(셰이더/캔버스).
- **L3 캐릭터·수레·파티클** — 일꾼(채굴 루프), 수레(레일 위, 수확 트윈), 반짝임.
  `assets/scene/miner.png`, `cart_empty.png`, `cart_full.png`
- **L4 UI 오버레이** — HUD·수익률·수확버튼·독.

규칙:
- **성향은 색이 아니라 글씨**로 표시(v1). 광산은 금색 한 톤 통일.
  - (v2 옵션: 성향별 색 스킨 — 흰 수정 1세트를 코드 틴트해 N색. 지금은 보류.)
- **수확 인터랙션**: 탭 → 수레가 레일 따라 금더미로 트윈 이동 → 합체 → 금더미 크기↑ + 숫자 카운트업. **실제 배당 데이터 기반.**
- 에셋은 **한 장에 굽지 말 것** — 배경/광맥/금더미/수레/일꾼 분리 유지(애니·교체용).

### HUD 구성 (확정)
- **좌상단**: 광맥 등급 `광맥 A · 88` / 그 아래 성향 칩 `성향 · 공격형`
  - 시장이 US면 등급 라벨이 `US 광맥 A · 91` 로 바뀜.
- **상단 중앙**: 시장 전환 토글 버튼 (아래 "시장 토글" 참고)
- **우상단**: 티어 엠블럼(작게) + `마스터 4` + 알림 점
- **금괴 칩**: **금괴 아이콘(이미지)** + 금액(누적 자산). 시장 통화 따라 `₩1.25M` / `$9,800`. "금괴"는 글씨 아님, 그림.
- **레벨 없음.** 티어는 `마스터 4 → 3 → 2 → 1` 식 디비전.
- **수익률 패널**(중앙 상단, 탭 가능): `▲ +18.4% / 평가수익 +₩1.72M`

### 보유 종목 (개별주) — 동적 데이터 리스트
- **종목은 절대 씬 아트에 굽지 않는다.** 보유 배열로 코드가 그리는 칩/리스트. → 매도=칩 빠지고 자동 정렬, 매수=칩 추가. (이게 sell/buy UI 문제의 해법)
- **홈(미리보기)**: 수익률 패널 바로 밑에 **상위 3개만** 칩으로(`삼성전자 +12.4%` 식). 넘으면 숨기고 `›` 표시.
- **펼침(전체)**: 수익률 패널/`›` 탭 → 바텀시트로 **현재 시장의 전 종목** 표시(이름·평가금액·수익률, 스크롤). 씬은 디밍.
- 수익률 색: 이익=초록, 손실=빨강.

### 시장 토글 (KR / US) — v1 포함
- 홈은 **한 번에 한 시장**만 본다(KR 또는 US). 상단 중앙 버튼으로 전환.
- 버튼 라벨: KR 보는 중 → `US 광맥 이동`, 누르면 US로 전환 후 버튼 → `KR 광맥 이동`.
- 전환 시 **씬(광산 그림)은 동일**, 데이터만 교체: 수익률·개별주·금괴 금액·**통화(₩↔$)**·등급 라벨(`광맥 A`↔`US 광맥 A`).
- **KR/US 완전 분리** — 각자의 광맥 등급·금더미·티어. 환율 합산 안 함. (통합 보기는 추후 별도 기능)

### 엠블럼
- 9등급(아이언~챌린저), **1디자인 색만 다름**(육각 메탈 + 수정). `assets/emblems/{tier}.png`
- HUD엔 작게. 등급업 연출 = 엠블럼 + 디비전 별 차오름.

## 6. 점수 / 등급 로직 (요약 — 상세 산식은 PDF §6)
- **성향 분류**: 변동성·섹터집중도·성장vs배당 비중·현금비중·종목수 → 4유형. 30일 이동평균 + 히스테리시스(잦은 전환 방지). **규모 무관.**
- **미래정합성(0–100)**: 테마정합 50% + 재무 30% + 수급 20%. 등급 매핑(예 90+ A). **종목별 기여도 투명 표기.**
- **티어 / XP**: 배치(스타일 중립, 상한 골드) → 등반(행동·비율·연속 XP). **액수 비례 금지, 손실 무강등.** 디비전 IV→I.
- **종목 미리보기(탐사)**: 종목명 입력 → 가상 비중으로 미래정합성 재계산 → 등급 변화(88→91). **"가정 시뮬·투자권유 아님" 고정.**

## 7. 화면 (5개, 하단 독)
- **광산(홈)**: 금광 씬 + HUD + 수익률 패널 + **개별주 상위 3개 미리보기** + **KR/US 시장 토글**. 감성·글랜스.
- **금고**: 매매·배당 자동 기록(영구 장부), 월/연 조회, 수확, Excel/PDF Export. **현재 시장 전 종목 리스트**(홈 펼침과 연결). ← v1 핵심
- **측량소**: 미래정합성 점수 분해 + 테마 서사.
- **탐사**: 종목 미리보기(what-if).
- **연구소**: 종목 심층 분석(재무·수급·이슈).

> **US 데이터 주의**: 홈의 KR/US 토글은 v1 UI에 포함하되, **미국 주식 시세·거래·배당 데이터 소스 확보 여부를 먼저 확인**할 것. 키움 해외주식 API 또는 대체 소스가 안 되면, v1은 US를 "준비 중(잠금)" 상태로 두고 KR만 실데이터로 가고, US 실연동은 v2로. (UI/토글 구조는 그대로 두고 데이터만 채우는 식.)

## 8. 예상 함정 (Pre-mortem — 상세 PDF §4)
1. **키움 과거 거래·배당 제공 범위** — 연동 단계서 **제일 먼저 확인**. 부족하면 "설치 후 기록" 명시 + CSV 임포트.
2. **수익률 계산이 키움 표시값과 일치해야** — 평단(가중평균)·실현/평가손익 구분. 산식 문서화.
3. **자본시장법** — 매수/매도 조언 금지.
4. **Railway stale uvicorn 캐시** — 배포 후 강제 재시작 체크.
5. 배당 정확성(배당락·세전후·환율), 수급 데이터 지연, 시세 지연, 보안(금융데이터).

## 9. 로드맵
- **v1**: 로그인 + 광산 홈(금색 씬, 성향 글씨) + 금고(키움 자동기록·수익률·배당·Export). ← 핵심 itch
- **v2**: 측량소(미래정합성) + 탐사. 점수/등급 로직 확정 선행.
- **v3**: 실시간 3D 씬(블렌더→Three.js), 수확 트윈/채굴 애니, (옵션)성향 색 스킨.

## 11. 빌드 계획 — 다음 작업 (Claude Code)
**Phase 1 (스캐폴드)**
- React+Vite + FastAPI + SQLite 프로젝트 `gwangmaek` 생성. Python 3.12.
- 아키텍처 먼저 제안 → 승인 후 코드.
- 3개 화면(광산/금고/측량소) **MOCK 데이터**로 동작. `kiwoom_client.py`는 stub(인터페이스만).
- 로컬 실행 가능 상태까지.

**Phase 2 (금고 = 핵심)**
- 키움 연동(앱키·토큰, 서버 보관). **과거 거래·배당 제공 범위부터 검증.**
- 매매·배당 자동 기록 → SQLite 영구 저장. 수익률 계산(키움 일치). Export.

**Phase 3 (점수/측량소·탐사)** → **Phase 4 (3D·애니)**

```

2) 화면 3개(광산 홈 / 금고 / 측량소)를 MOCK 데이터로 동작하게.

```
