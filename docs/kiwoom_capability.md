## 1. 과거 매매(체결) 내역 조회 가능 기간

| API 코드 | 이름 | 조회 가능 기간 |
|---|---|---|
| **kt00009** | 계좌별주문체결내역상세 | **최근 2개월(약 60일) 고정** |
| ka10076 | 체결요청 | 당일 체결만 |
| ka10072 | 일자별종목별실현손익_일자 | 날짜 지정 (기간 한도 미공개) |
| ka10073 | 일자별종목별실현손익_기간 | strt_dt~end_dt 범위 (기간 한도 미공개) |
| ka10074 | 일자별실현손익 | strt_dt~end_dt 범위 (기간 한도 미공개) |

**핵심 제약: kt00009는 최근 2개월만.** kiwoom-restful 패키지 문서에 "계좌 체결내역 데이터 (최근 2달만)"으로 명시.  
→ 2개월 이전 과거 매매는 API로 소급 불가. **CSV 임포트 폴백 필수.**

## 2. 배당 수령 내역 제공 여부

**없음.** 공개된 모든 ka/kt API 목록(33개 계좌 관련 엔드포인트)에 배당금 수령내역 조회 엔드포인트가 존재하지 않음.

- ka 계열 계좌 API: 체결/실현손익/미체결/수익률 위주
- kt 계열 계좌 API: 예수금/평가잔고/체결잔고 위주
- 배당 관련 항목명 어디에도 없음

**→ 배당 내역은 반드시 폴백 경로 필요.**

## 3. 평가금액·평단가·실현손익 관련 조회 항목

### 실시간 잔고/평가 (연동 후 즉시 활용 가능)

| API 코드 | 이름 | 포함 정보 |
|---|---|---|
| **kt00004** | 계좌평가현황요청 | 계좌 전체 평가금액, 총평가손익, 수익률 |
| **kt00005** | 체결잔고요청 | 종목별 보유수량, 평가금액, 손익 |
| **kt00018** | 계좌평가잔고내역요청 | 종목별 평단가·평가금액·실현손익 상세 |
| ka10085 | 계좌수익률요청 | 보유종목별 수익률 |
| ka10077 | 당일실현손익상세 | 당일 매도 실현손익 |
| ka10073 | 일자별종목별실현손익_기간 | 기간별 종목별 실현손익 누적 |

### 예상 응답 필드명 (공식 가이드 로그인 필요 — 연동 시 실확인)

공식 필드명은 로그인 없이 확인 불가. 키움·한투 API 관례상 아래 패턴으로 추정:

| 의미 | 예상 필드명 |
|---|---|
| 평단가(가중평균 매수단가) | `pchs_avg_pric` 또는 `avg_pchs_price` |
| 평가금액 | `evlu_amt` |
| 평가손익 | `evlu_pfls_amt` |
| 수익률 | `evlu_pfls_rt` |
| 실현손익 | `rlzt_pfls` 또는 `rlzt_pfls_amt` |
| 보유수량 | `hldg_qty` |

**→ kt00018이 핵심.** 현재 보유 종목의 평단·평가 일괄 조회 가능.

## 4. 조회 한도 / Rate Limit

| 구분 | 수치 |
|---|---|
| TR당 안전 속도 | **약 1 req/s** |
| 순간 버스트 허용 | TR당 약 2건 |
| 초과 시 응답 | HTTP 429 |
| 토큰 유효기간 | 24시간 |

- TR(API 코드)별로 독립 적용 (ka10073 따로, kt00018 따로)
- 429 발생 시 지수 백오프 후 재시도 구현 필요
- 모의투자/실전투자 서버 분리 (`is_paper_trading` 파라미터)

## 6. 폴백 설계 — 부족한 데이터 보완 전략

### 전략: "즉시 누적 시작 + 소급 CSV 임포트"

```
[A] 키움 REST 연동 이후 체결 → DB 자동 저장 (롤링 누적)
[B] 과거 체결 (2개월 이전) → 사용자가 HTS에서 CSV 내보내기 → 앱 임포트
[C] 배당 내역 → 사용자가 HTS에서 CSV 내보내기 → 앱 임포트
```

### 구현 포인트

1. **자동 누적(A)**: 매일 kt00009 풀링 → trades 테이블에 upsert (중복 방지: ord_no + exec_dt 복합 PK)
2. **CSV 임포트(B·C)**:
   - FastAPI `POST /api/import/trades` — 멀티파트 CSV 업로드
   - FastAPI `POST /api/import/dividends` — 배당 CSV 업로드
   - 키움 HTS CSV 컬럼명 매핑 레이어 별도 작성 (`importers/kiwoom_csv.py`)
3. **UI**: 금고 화면 상단에 "과거 내역 가져오기" 버튼 (CSV 파일 선택 → 미리보기 → 저장)
4. **데이터 소스 태깅**: 모든 레코드에 `source: api | csv_import | manual` 컬럼 → 나중에 신뢰도 표시 가능

### HTS에서 내보낼 수 있는 CSV (키움 HTS 기준)

| 파일 | 경로(HTS) | 주요 컬럼 |
|---|---|---|
| 체결내역 | 주문/잔고 → 체결내역 → 엑셀저장 | 체결일, 종목코드, 종목명, 매수/매도, 체결수량, 체결단가, 수수료 |
| 배당내역 | 잔고 → 배당금 입금내역 → 엑셀저장 | 입금일, 종목명, 배당금액, 세금 |

> HTS 메뉴 경로는 버전마다 다를 수 있음. 실사용 시 재확인.
