# 측량소 점수 체계 v2 — 2모드 재설계 (설계 제안)

> 작성일: 2026-06-18 · 상태: **제안(검토 대기)** · 승인 전 구현 금지 (CLAUDE.md 하드룰 §4)
> 대체 대상: `docs/scoring_design.md`(v1, 단일 "성장테마 화이트리스트" 렌즈)

## 0. 왜 바꾸나 (문제 정의)

v1 측량소의 두 가지 결함:

1. **주관 강요** — `themes.json`의 "성장테마 화이트리스트(AI·로봇·SMR·전력·2차전지)"는 **큐레이터(개발자)의 주관**이다. 사용자에게 "이게 성장이다"를 강요하는 구조 → CLAUDE.md 정신(스타일 중립, "하나의 렌즈일 뿐")과 충돌.
2. **테마 매칭 미작동** — `ka90001`(키움 테마 API) 경로 미확인으로 `get_theme_index()`가 빈 dict 폴백 → **로봇주(에스피지)조차 theme=0**. 즉 "성장 렌즈"가 실제로는 작동 안 함. 현재 점수는 사실상 재무+수급만으로 굴러가며 theme=0이 전 종목을 끌어내림.

→ **결론**: 주관적 테마를 점수의 디폴트 축에서 빼고, 테마는 **사용자가 켜는 옵션 렌즈**로 분리한다.

## 1. 두 모드 개요

| | ① 기본 모드 (디폴트) | ② 테마 모드 (사용자 설정 시) |
|---|---|---|
| 성격 | "내 포트의 **객관 펀더멘털** 스냅샷" | "내가 고른 **방향성**에 내 포트가 얼마나 정렬됐나" |
| 렌즈 정의자 | 없음 (검증가능 데이터만) | **사용자 본인** |
| 점수 축 | 재무건전성 + 수급 + 시장신뢰 | 테마 정렬도 × (정렬 종목 펀더멘털) |
| 진입 | 처음 들어온 모두가 봄 | "성장테마 설정하기" 누른 사용자만 |
| 복귀 | — | "내 성장테마 삭제하기" → ①로 |
| HUD 등급 라벨 | `광맥 A · 88` | `AI테마 정렬 A · 88` (테마명 부기) |

**불변 하드룰(둘 다)**: 투자권유 아님 · 점수 낮음 ≠ 나쁨 · 매수/매도 조언 금지 · 티어/XP와 무관. (§5)

## 2. 데이터 확보 가능성 점검 (게이트 — 먼저)

설계의 첫 원칙: **확보 가능한 데이터로만 짠다.** 현재 코드 기준 실제 가용 현황:

| 지표 | 소스 | 상태 | 비고 |
|---|---|---|---|
| 매출·영업이익 **성장률(YoY)** | DART `fnlttSinglAcntAll` | ✅ **작동 확인** (`dart_client.get_financial_scores`) | 연간보고서, 전년 폴백 구현됨 |
| **수익성·안정성**(영업이익률, 흑자지속, 부채비율) | DART `fnlttSinglAcntAll` (전체 계정) | ⚠ **부분** — 흑자지속만 사용 중. 영업이익률·부채비율은 동일 응답에서 **추가 파싱만 하면 가능** | `_find_account`에 계정 추가 |
| **수급**(외인+기관 순매수비율 20일) | 키움 `ka10059` `/api/dostk/stkinfo` | ✅ **작동 확인** (`get_supply_scores`) | 비율 정규화로 규모 중립 |
| **시가총액·거래대금·PER/PBR/ROE** | 키움 `ka10001` `/api/dostk/stkinfo` | ⚠ **미검증** — 응답 필드 확인 필요 (`probe_ka10001.py` 이미 존재, **실행 후 필드 확정**이 게이트) | 메모리: `sale_amt`, `bus_pro` 등 포함 추정 |
| **테마 ↔ 종목 매핑** | 키움 `ka90001`/`ka90002` | ❌ **경로 미확인** (1504) | → §4 대체 전략 필수 |
| 업종(KSIC) 분류 | DART `company.json` (`induty_code`) | ⚠ 미연동, **확보 가능**(무료) | 테마 매칭 객관 폴백용 |

**선결 작업 2개 (승인 시):**
1. `probe_ka10001.py` 실행 → 시총·거래대금·밸류에이션 **실제 필드명 확정** (기본 모드 "시장신뢰" 축의 가용성 결정).
2. DART `company.json` 연동 검토 → `induty_code`(업종) 확보 가능 여부 확인 (테마 매칭 폴백).

> 키·시크릿은 모두 `.env`, 서버 보관, git·프론트 노출 금지 (하드룰 §4).

## 3. ① 기본 모드 — 객관 펀더멘털 렌즈 (상세)

### 3.1 설계 철학 (스타일 중립 충돌을 정면으로 다룸)

⚠ **핵심 긴장**: CLAUDE.md §4는 "점수는 스타일 중립, 특정 철학 우대 금지"라고 못박는다. 그런데 "성장률"을 점수에 넣으면 **성장주 우대**, "시가총액"을 넣으면 **대형주 우대**가 된다. 사용자 요청(매출/이익 성장 + 안정성 + 규모)을 그대로 넣으면 이 하드룰과 부딪힌다.

**해소 원칙 — "사실 vs 우열"의 구분:**
- 기본 모드는 **"이 스타일이 우월하다"를 말하지 않는다.** 각 종목의 **검증가능한 기업 사실**(재무 상태, 자금 흐름, 시장 참여도)을 그대로 비춘다.
- 단일 축(성장만, 규모만)이 점수를 지배하지 않도록 **상호 상쇄되는 다축 구성**으로 중립성을 확보한다. (성장↑ 점수와 안정성↑ 점수를 함께 둠 → 고성장 적자기업과 저성장 흑자기업이 서로 다른 축에서 득점)
- **모든 화면에 "이건 우열 평가가 아니라 사실 스냅샷"** 문구 고정.

### 3.2 종목 서브점수 구성 (제안)

각 종목 `i`에 대해 0~100 서브점수 3개:

```
fundᵢ = 0.5·health_i + 0.3·supply_i + 0.2·trust_i      (가용 항목만, N/A 재정규화)
```

| 축 | 가중 | 구성 (모두 검증가능 데이터) | 소스 | 상태 |
|---|---|---|---|---|
| **재무건전성** `health` | 50% | 수익성(영업이익률·흑자지속) + 안정성(부채비율) + 성장성(매출·영업이익 YoY) — **세 하위축 균형** | DART | 성장성 ✅ / 수익성·안정성 ⚠ 파싱추가 |
| **수급** `supply` | 30% | 외인+기관 20일 순매수비율 → tanh | 키움 ka10059 | ✅ |
| **시장신뢰** `trust` | 20% | 시가총액·평균거래대금(유동성·시장 관심) | 키움 ka10001 | ⚠ 필드검증 후 결정 |

- `health` 내부(제안): `0.4·수익성 + 0.3·안정성 + 0.3·성장성`. → 성장 단일 지배 방지(성장 비중 = 전체의 0.5×0.3 = **15%**로 제한).
- `trust`는 **시총·거래대금을 로그 스케일 백분위**로 0~100 (대형주 자동 만점이 아니라 분포 내 상대 위치). 필드 미확보 시 **이 축 드롭 → health/supply만으로 재정규화** (N/A 규칙).
- 포트 총점: `S = Σ wᵢ·fundᵢ`, 기여도 `cᵢ = wᵢ·fundᵢ` (Σcᵢ = S, v1의 정확 분해 구조 계승).

### 3.3 산정 불가 종목 (현금/ETF/리츠/스팩)
v1 §6 규칙 그대로: N/A 축은 **감점 아님 → 가중치 재정규화**, `flags`로 투명 표기. 현금은 분모 포함·점수 0 기여·"현금 비중" 별도 표기.

## 4. ② 테마 모드 — 방향성 정렬도 (상세)

### 4.1 정의
"**사용자가 직접 고른 방향**(예: AI, 로봇, 우주)에 내 포트가 얼마나 정렬됐고, 그 정렬 종목들의 펀더멘털이 받쳐주나."

### 4.2 점수 산식 (정렬 × 품질)

```
정렬도 alignᵢ  = 1  (종목 i가 사용자 테마에 매칭) | 0 (미매칭)
품질    qualᵢ   = §3 의 fundᵢ  (기본 모드 펀더멘털 재사용)
종목점수 sᵢ     = alignᵢ · qualᵢ          ← 정렬 안 되면 0 기여, 정렬되면 펀더멘털만큼

테마 정렬 점수 S = Σ wᵢ · sᵢ
```

**왜 "정렬 × 품질"인가**: 단순 정렬 비중(%)만 쓰면 "테마 종목을 아무거나 많이 담으면 만점"이 된다. 펀더멘털로 가중하면 "방향도 맞고 기업도 받쳐주는 정도"가 된다. **품질 축이 §3을 재사용**하므로 코드·데이터가 일관.

### 4.3 화면에는 **두 숫자**를 분리 노출 (투명성)
- **정렬도**: `Σwᵢ·alignᵢ` = "내 평가금액 중 이 테마에 속한 비중 %".
- **정렬 종목 펀더멘털 평균**: 매칭 종목들의 `fundᵢ` 가중평균.
- **헤드라인 점수 S** = 위 둘의 결합(=정렬×품질). 사용자가 "비중이 낮아서 낮나, 종목이 약해서 낮나"를 구분 가능.

### 4.4 테마 정의(사용자 입력) 방식
- **프리셋 선택**(권장 디폴트): 우리가 제공하는 **방향성 보따리 목록**(AI·반도체 / 로봇 / SMR·원전 / 전력인프라 / 2차전지 / 우주·방산 / 바이오 …)에서 1개 이상 선택. — 단, **이건 "추천 성장 분야"가 아니라 "자주 쓰는 렌즈 템플릿"**으로 표기(주관 강요 회피). 기존 `themes.json`을 이 프리셋 소스로 재활용.
- **커스텀 입력**(옵션): 사용자가 키워드 직접 입력(예: "수소"). → §4.5 매칭 엔진이 처리.
- **다중 테마**: 합집합(OR) 매칭. 종목이 어느 테마에 걸렸는지 칩으로 표기.

## 5'. 테마 종목 매칭 — ka90001 없이 (핵심 난제)

`ka90001`이 안 되므로 **3단 폴백 매칭 엔진**을 제안. 각 매칭에 **근거(source)를 함께 저장·표시**(투명성).

| 단계 | 방법 | 데이터 | 신뢰도 | 비고 |
|---|---|---|---|---|
| **A. 큐레이션 바스켓** | 테마별 **명시적 종목코드 리스트**를 우리가 유지 (`theme_baskets.json`) | 자체 관리 | 높음 | 에스피지=로봇 같은 "이름으로 안 잡히는" 종목 해결. **사용자가 테마를 고른 뒤의 구성종목**이므로 v1의 "테마 자체 강요"와는 성격 다름 |
| **B. 업종(KSIC) 매핑** | DART `induty_code` → 테마 매핑표 | DART `company.json` | 중간 | 객관·검증가능하나 거침(교차테마 못 잡음). A의 보강 |
| **C. 키워드 매칭** | `themes.json` 키워드 ↔ (종목명 + 업종명) | 보유 데이터 | 낮음 | 최후 폴백. "삼성전자↔반도체"류만 잡힘 |

- **매칭 우선순위**: A > B > C. 매칭 결과에 `matchSource: "basket"|"ksic"|"keyword"` 부착 → 화면에 "왜 매칭됐는지" 노출.
- **바스켓 운영 원칙(하드룰)**: 바스켓은 "이 종목을 사라"가 아니라 "이 종목이 이 분야에서 사업한다"는 **사실 분류**. 편입 근거(사업영역)를 함께 기록. **종목 추천으로 읽히지 않도록** 문구 고정.
- **에스피지 사례 해결**: 바스켓(A)에 `로봇: ["058610", …]` 등록 → ka90001 없이도 로봇 정렬로 정확히 잡힘.

> ⚠ 바스켓도 결국 누군가 구성종목을 고른다. v1과의 결정적 차이: **(1) 사용자가 그 테마를 능동적으로 켰고, (2) 매칭 근거를 투명 표기하며, (3) 기본 모드(디폴트)는 바스켓에 전혀 의존하지 않는다.** 주관은 "옵션 렌즈" 안에만 격리된다.

## 5. 하드룰 (두 모드 공통 — 위반 금지)

CLAUDE.md §4를 두 모드에 맞게 구체화:

1. **투자권유 아님** — 모든 응답/화면에 disclaimer 고정.
2. **점수 낮음 ≠ 나쁨**:
   - 기본 모드: 낮은 점수 = "수급이 약하거나 성장/수익 지표가 낮다"는 **중립적 사실 스냅샷**. 우열 아님.
   - 테마 모드: 미매칭/낮음 = "이 **선택한 테마**에 안 속하거나 덜 정렬됨". 다른 전략(배당·가치)은 애초 이 렌즈로 측정 안 됨.
3. **매수/매도 조언 금지** — 특히 테마 모드: "이 테마 종목을 사라/늘려라" **절대 금지**. 테마 모드는 *내가 고른 렌즈로 내 포트를 비추는 것*일 뿐, 종목 추천 아님.
4. **티어/XP 무관** — 이 점수는 측량소 전용. 손실 강등·액수 비례 없음(§4 불변).
5. **투명성** — 종목별 기여도(cᵢ), 매칭 근거(matchSource), 데이터 기준일(asOf), N/A 플래그 모두 노출.
6. **UI 중립 톤** — 낮은 점수를 빨강(=경고)으로 칠하지 않음. "사실 스냅샷"임을 환기하는 문구 동반.

## 6. 모드 전환 UX

```
[측량소 진입]
   └─ 사용자 테마 설정 있음? ──No──▶ ① 기본 모드 (객관 펀더멘털)
                                        상단: "객관 펀더멘털 렌즈" 배지 + 점수/등급
                                        [+ 내 성장테마 렌즈 만들기] 버튼
                                        │ 탭
                                        ▼
                                   [테마 선택 시트] 프리셋 다중선택 / 커스텀 입력
                                        │ 확정
                                        ▼
            ──Yes─────────────────▶ ② 테마 모드 (정렬도 × 품질)
                                        상단: "AI·로봇 정렬 렌즈" 배지 + 정렬도 + 점수
                                        본문: 매칭 종목 칩(근거 표기) / 미매칭 종목 "중립"
                                        [내 성장테마 삭제하기] ──▶ ① 복귀
```

- **저장**: 사용자×시장(KR/US)별 테마 선택을 SQLite에 영속. (KR/US 분리 — CLAUDE.md §5)
- **HUD 연동**: 홈 광맥 등급 라벨이 활성 모드를 반영(`광맥 A·88` ↔ `AI테마 A·88`). 씬은 불변(금색 통일).
