## 2. 레이아웃
- **상단 고정 헤더**(Lv·EXP·접속일차·탭칩·총 수익률): `position: fixed` 상단.
  바로 아래 dark→transparent 그라데이션 스크림 → 뒤 광산이 비쳐도 글씨 가독성 확보.
- **배경 plate**: 헤더 밑 ~ 하단탭 위 영역에 **세로(height) 기준 scale-to-fit**, 가로 중앙정렬.
  좌우 남는 공간은 검은 letterbox 허용(plate 테두리가 검정이라 티 안 남).
  헤더 밑 여백 더 필요하면 plate 위에 검은 padding만 추가(재생성 불필요).
- **하단 고정 탭바**: 광산·금고·측량소·탐사·소식.

## 3. 일꾼 배치 좌표 (★핵심)
- 기준: **렌더된 plate 이미지**의 폭/높이에 대한 비율(%). (화면이 아니라 plate 기준)
- 앵커: **발-중앙 (bottom-center)**. 즉 `left = cx*plateW - spriteW/2`, `top = foot_y*plateH - spriteH`.
- 스프라이트 높이 = `h_frac * plateH`.

| 챔버 | 종목(예시 매핑) | 스프라이트 | cx | foot_y | h_frac | flipX |
|---|---|---|---|---|---|---|
| 1 (상단) | 최대보유 | miner_1_carry | 0.540 | 0.205 | 0.070 | 아니오 |
| 2 | 2번 | miner_2_sawing | 0.385 | 0.470 | 0.061 | 아니오 |
| 3 | 3번 | miner_3_sorting | 0.560 | 0.715 | 0.065 | 아니오 |
| 4 (하단) | 4번 | miner_4_pickaxe | 0.500 | 0.865 | 0.072 | 아니오 |

- **flipX = C4만 예** (`scaleX(-1)`): C4는 **왼쪽 광맥** 앞에 배치. 반전해서 왼쪽(광맥)을 마주보게. 오른쪽 광맥은 공간이 좁아서 몸이 벽에 붙으므로 왼쪽 광맥 사용. C1은 반전 없이 오른쪽 금맥에서 금 줍기.
- 종목 라벨칩(종목명·등락%·평가금액)은 각 챔버 상단 빈 바위벽 쪽에 배치(미세조정 in-app).
- 광맥 발광 포인트 = 각 챔버 금맥 위.

## 7. 줌 시스템 (V2 구조 — 줌인 ↔ 줌아웃)

### 개념
홈 화면은 **핀치 줌**으로 두 상태를 오감.
- **줌인**: 한 챔버를 크게 — 일꾼·금맥 디테일 (현재 4챔버 plate 화면이 이 상태)
- **줌아웃**: 내 광산 **전체 수직 갱도**를 위에서 아래로 조망 — 보유 종목 전부가 챔버로 연결되어 보임
- 종목 수가 적으면 줌인 상태로 시작, 많아질수록 줌아웃으로 전체가 보이게.
- swipe 페이지네이션 → **핀치 줌 + 수직 스크롤**로 대체.

### 챔버 모듈화 (필수) — C안: 다층 격자
- 구조: **4챔버 격자 = 1층(floor)**. 종목 5개째부터 2층 시작, 9개째부터 3층. (한 층 = 2열 × 2행 지그재그 격자)
- 층이 **수직으로 쌓임** → 줌아웃하면 깊은 다층 광산. 격자(가로 채움) + 수직 깊이(층) 동시 확보.
- 모듈 단위:
  - 챔버 본체 1칸 (좌/우 변형 2종) + 층 내 격자 배치
  - 층-사이 연결부(수직 갱도/사다리·레일) 조각 — 층이 늘면 이어붙임
- 기존 4챔버 plate(`gwangmaek_plate_clean.png`)는 **1층 프리셋(줌인 고퀄)** 으로 그대로 재활용.
- 빈 슬롯(보유보다 칸이 남으면): **불 꺼진 어두운 챔버**(일꾼 없음).

### 줌아웃 화면 사양 (목업: zoom_versions.png 의 C안)
- **층 단위로 4챔버 격자**(2열×2행 지그재그), 층끼리는 수직 갱도/사다리로 연결.
- 각 챔버: 금맥 발광(작게) + 일꾼 실루엣 + 박스 없는 라벨(종목명 세리프·본색 + 수익률).
- 층 사이/아래로 갈수록 depth haze(어두워짐) = 깊이감.
- 보유 종목보다 칸이 남으면: 빈 챔버는 **불 꺼진 어두운 상태**(일꾼 없음).
- 줌인: 한 챔버(또는 한 층) 크게. 줌아웃: 다층 전체 조망. 핀치/우상단 줌 토글.
- 종목 적으면 1층만(줌인 상태로 시작), 늘수록 줌아웃되며 다층이 드러남.

### 정체성 가드 (레퍼런스와 차별)
- AI agent 앱들의 **네온 평면 격자** 따라가지 말 것. 광맥은 **따뜻한 painterly 수직 갱도**.
- "일하면 자동 증식 / 도파민 후킹" 같은 **앱 주도 메커니즘은 배제** (§12). **챔버 증식 트리거 = 사용자의 능동적 행위(종목 매수)**. 레퍼런스 AI agent 앱이 "자동화 작업 1개 추가 → 방 1개 증식"인 것과 동일 구조로, 광맥은 **종목 1개 매수 → 챔버 1개 증식**. 매도하면 챔버 제거(또는 불 꺼진 빈 챔버로).
- 레퍼런스에서 가져오는 건 **줌(전체↔디테일) 하나뿐**. 수직 깊이 = 장기 보유 메타포.

### 라벨 최종 스타일 (label_directing_mockup.svg 참고)
- 닫힌 박스 제거. 글자 뒤 soft radial shadow만.
- 종목명: 따뜻한 본(bone)색 세리프, 음각(letterpress) 느낌.
- 수익률: 채도 낮춘 초록/빨강 + 작은 광물 마름모. ▲▼ 대신.
- **평가금액 표시 안 함** (상단 총평가금액으로 충분). 종목명 + 수익률만.
- 라벨은 해당 챔버의 금맥/일꾼 옆에 앵커 (가장자리 정렬 X).
